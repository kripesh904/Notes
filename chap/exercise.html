1. Definitions

a. Variable

A variable is a named memory location used to store data that can be modified during the execution of a program. It allows dynamic assignment of values based on conditions, user inputs, or calculations done by the system.

b. Expression

An expression is a valid combination of variables, constants, operators, and functions that produces a single value when evaluated. It is essential for performing operations, assigning values, and controlling flow within a program.

c. Identifier

An identifier is the name assigned to elements like variables, functions, arrays, or structures in a program. It must start with a letter or an underscore and should not match any of the language's reserved keywords.

d. Keyword

A keyword is a reserved word that has a predefined meaning in the programming language. These words are used for defining the structure and syntax and cannot be used for variable or function names.

e. Constant

A constant is a value that does not change during the execution of a program. Once declared and assigned, its value remains fixed and is used to prevent accidental changes to critical data.

f. Operator

An operator is a symbol used to perform operations on variables and values. It includes arithmetic, logical, relational, bitwise, assignment, and other types, depending on the language's capabilities.

g. Data Type

A data type defines the kind of value a variable can hold, such as integers, floats, characters, or strings. It determines the size, range, and operations that can be performed on the variable.

h. Token

A token is the smallest unit of a program that the compiler recognizes. Examples include keywords, identifiers, constants, operators, and punctuation symbols, which together form the basic syntax of the language.

i. Compiler

A compiler is a software tool that translates source code written in a high-level programming language into machine code. It checks the code for syntax errors and generates executable files that can run independently.

j. Interpreter

An interpreter is a program that reads and executes code line by line, converting it to machine code at runtime. Unlike a compiler, it does not produce a separate executable file but runs the code directly.

k. Header File

A header file is a file containing declarations of functions and macros. It is included in the source file using the #include directive and provides access to standard or custom library functions.

l. Bitwise Operator

Bitwise operators are used to perform operations on individual bits of data. These include AND, OR, XOR, NOT, left shift, and right shift, and they are mainly used for low-level programming and performance optimizations.

m. Type Conversion

Type conversion is the process of converting a variable from one data type to another. It can be implicit (automatic) or explicit (casted by the programmer) and is used when combining different data types in expressions.

2. Short Questions

1. What is the objective of the IT Policy 2072?

The main objective of the IT Policy 2072 in Nepal is to transform the nation into a knowledge-based society by utilizing information technology to drive development and economic growth. It aims to enhance IT infrastructure and accessibility across the country.

2. What is malware? Write any two types of malware.

Malware is malicious software designed to damage, disrupt, or gain unauthorized access to a computer system. It includes viruses, worms, and other harmful programs.

Two types of malware are:

  • Virus: A program that attaches itself to other files or programs, spreading when those files are executed.
  • Worm: A self-replicating program that spreads across networks and systems without needing to attach to another file.

3. What is cyber law?

Cyber law refers to legal regulations that govern the use of the internet and digital technology. It deals with issues such as cybercrimes, digital transactions, data privacy, intellectual property, and the legal aspects of e-commerce and online activities.

4. Write any two features of a digital signature.

A digital signature ensures the authenticity and integrity of electronic messages or documents.

Two features are:

  • Authentication: It verifies the identity of the sender of the message or document.
  • Integrity: It ensures that the content has not been altered during transmission.

5. Define computer ethics.

Computer ethics is the set of moral principles and standards that guide the use of computers, software, and technology. It includes issues like privacy, security, intellectual property, and the responsible use of digital resources.

3. Long Questions

a. What are the different types of errors in programming? Explain with examples.

Errors in programming can be categorized into three types: syntax errors, semantic errors, and runtime errors.

Syntax Errors: These errors occur when the code violates the syntax rules of the programming language. The compiler detects these errors and prevents the program from compiling. 

            printf("Hello, World!")  // Missing semicolon


Semantic Errors: These errors happen when the code is syntactically correct but doesn’t perform as expected due to logical flaws. 

            int a = 5, b = 0;
            printf("%d", a / b);

Runtime Errors: These errors occur when the program is running. They usually result from invalid operations such as dividing by zero, accessing memory incorrectly, or using null pointers. 

            int *ptr = NULL;
            printf("%d", *ptr);


b. Explain the different ways of adding CSS to an HTML file.

CSS can be added to an HTML file in three main ways: inline, internal, and external.

Inline CSS: This method involves adding CSS rules directly within HTML tags using the style attribute. It is useful for small, specific styling but not ideal for large projects. 

        <p style="color: blue;">This is blue text.</p>


Internal CSS: CSS is defined within a style block inside the <head> section of the HTML document. This method is used when the styles apply to a specific page. 

        <html>
          <head>
            <style>
              p {
                color: green;
              }
            </style>
          </head>
          <body>
            <p>This is green text.</p>
          </body>
        </html>


External CSS: CSS is written in a separate .css file and linked to the HTML file using the <link> element. This method is best for large projects, as it separates content from design and allows for code reuse. 

        <!-- index.html -->
        <head>
          <link rel="stylesheet" href="style.css">
        </head>
        
        <!-- style.css -->
        p {
          color: red;
        }


c. What are the generations of computers? Discuss their key features and examples.

The history of computers is divided into five generations based on the technology used in their design and development.

First Generation (1940–1956): First-generation computers used vacuum tubes for circuitry, which made them large, slow, and power-hungry. They required a lot of space and generated heat, which made them difficult to maintain. The input was typically via punched cards, and output was through printouts. An example is the ENIAC (Electronic Numerical Integrator and Computer).

Second Generation (1956–1963): Transistors replaced vacuum tubes in second-generation computers, making them smaller, faster, and more reliable. They consumed less power and produced less heat. High-level programming languages like COBOL and FORTRAN were introduced. An example is the IBM 1401.

Third Generation (1964–1971): Integrated circuits (ICs) replaced transistors, leading to more powerful and compact computers. Third-generation computers could handle more complex tasks and supported multitasking. High-level languages like FORTRAN and COBOL became more widespread.

Fourth Generation (1971–Present): Microprocessors were introduced, which placed all components of a computer’s CPU on a single chip. This revolutionized the computer industry, leading to personal computers (PCs). Graphical user interfaces (GUIs), the internet, and high-level programming languages became widespread. Examples include the Apple II and IBM PC.

Fifth Generation (Present and Beyond): The focus of fifth-generation computers is on artificial intelligence (AI), machine learning, and robotics. These systems aim to simulate human intelligence and perform tasks that require reasoning and decision-making. Examples include AI-driven systems like virtual assistants and autonomous vehicles.

1. Program to Check Whether the Input Number is Prime

        #include <stdio.h>

        int main() {
            int i, n, c = 0;
            printf("Enter any number : ");
            scanf("%d", &n);
            for(i = 2; i < n; i++) {
                if(n % i == 0) {
                    c++;
                }
            }
            if(c == 0) {
                printf("Prime");
            } else {
                printf("Composite");
            }
            return 0;
        }



2. Program to Find Factorial of an Input Number

        #include <stdio.h>

        int main() {
            int i, n, a = 1;
            printf("Enter any number : ");
            scanf("%d", &n);
            for(i = 1; i <= n; i++) {
                a = a * i;
            }
            printf("Factorial = %d", a);
            return 0;
        }



3. Program to Check Whether the Input Number is Palindrome

        #include <stdio.h>

        int main() {
            int i, n, a, r = 0;
            printf("Enter any number : ");
            scanf("%d", &n);
            a = n;
            while(n != 0) {
                i = n % 10;
                r = r * 10 + i;
                n = n / 10;
            }
            if(a == r) {
                printf("Palindrome");
            } else {
                printf("Not a Palindrome");
            }
            return 0;
        }



4. Program to Generate Fibonacci Series

        #include <stdio.h>

        int main() {
            int i, n, a = 0, b = 1, c;
            printf("Enter any number : ");
            scanf("%d", &n);
            for(i = 1; i <= n; i++) {
                c = a + b;
                printf("%d ", a);
                a = b;
                b = c;
            }
            return 0;
        }



5. Program to Check Whether the Input Number is Perfect

        #include <stdio.h>

        int main() {
            int i, n, sum = 0;
            printf("Enter any number : ");
            scanf("%d", &n);
            for(i = 1; i <= n / 2; i++) {
                if(n % i == 0) {
                    sum = sum + i;
                }
            }
            if(sum == n) {
                printf("Perfect");
            } else {
                printf("Not Perfect");
            }
            return 0;
        }



6. Program to Print Pyramid Pattern

#include <stdio.h>

int main() {
    int n = 5, i, j, space;

    for(i = 1; i <= n; i++) {
        for(space = 1; space <= n - i; space++) {
            printf(" ");
        }

        for(j = 1; j <= 2 * i - 1; j++) {
            printf("*");
        }

        printf("\n");
    }

    return 0;
}



7. Program to Sum Main Diagonal Elements of a Square Matrix

#include <stdio.h>

int main() {
    int n, i, j, sum = 0;

    scanf("%d", &n);

    int matrix[n][n];

    for(i = 0; i < n; i++) {
        for(j = 0; j < n; j++) {
            scanf("%d", &matrix[i][j]);
        }
    }

    for(i = 0; i < n; i++) {
        sum += matrix[i][i];
    }

    printf("%d\n", sum);

    return 0;
}

9. Program to Sum Even and Odd Elements of a Square Matrix

#include <stdio.h>

int main() {
    int n, i, j, evenSum = 0, oddSum = 0;

    scanf("%d", &n);

    int matrix[n][n];

    for(i = 0; i < n; i++) {
        for(j = 0; j < n; j++) {
            scanf("%d", &matrix[i][j]);

            if(matrix[i][j] % 2 == 0) {
                evenSum += matrix[i][j];
            } else {
                oddSum += matrix[i][j];
            }
        }
    }

    printf("%d\n", evenSum);
    printf("%d\n", oddSum);

    return 0;
}
